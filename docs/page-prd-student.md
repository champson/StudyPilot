# AI高考伴学教练 学生端页面级 PRD

## 文档信息

**文档类型**  
学生端页面级需求规格说明书

**适用阶段**  
UI 设计 / 前端开发 / 接口联调

**技术栈**  
Next.js 14+ (App Router) + TailwindCSS + TypeScript

**产品定位**  
面向上海高中生的 AI 伴学教练，小范围自用（5人以内），电脑端优先，兼顾手机端

**关键约束**
- 首期两种学习模式：工作日跟学 + 周末复习，由学生手动切换
- 错题再次召回是 P0 功能
- 任务完成状态区分：已进入 → 已执行 → 已完成
- 掌握度双层：知识点掌握状态(5档) + 学科风险状态(4档)
- 5 门验收学科：语文、数学、英语、物理、化学

### 核心组件规格

#### 1. 步骤指示器 (StepIndicator)

| 属性 | 规格 |
|-----|------|
| 组件名称 | `StepIndicator` |
| 数据来源 | 本地状态 |
| 展示字段 | `currentStep`, `totalSteps`, `stepLabels[]` |
| 交互行为 | 点击已完成步骤可返回编辑 |
| 状态变化 | 未完成(灰色) → 当前(主色) → 已完成(绿色) |

#### 2. 年级选择器 (GradeSelector)

| 属性 | 规格 |
|-----|------|
| 组件名称 | `GradeSelector` |
| 数据来源 | 静态配置 |
| 展示字段 | `gradeOptions: ['高一', '高二', '高三']` |
| 交互行为 | 单选，选中后高亮显示 |
| 状态变化 | 未选中 → 选中(主色边框+背景) |
| 必填校验 | 是 |

#### 3. 教材版本选择器 (TextbookSelector)

| 属性 | 规格 |
|-----|------|
| 组件名称 | `TextbookSelector` |
| 数据来源 | API: `GET /api/v1/config/textbook-versions` |
| 展示字段 | `versionName`, `subjects[]` |
| 交互行为 | 下拉单选 |
| 必填校验 | 是 |

#### 4. 排名输入 (RankInput)

| 属性 | 规格 |
|-----|------|
| 组件名称 | `RankInput` |
| 数据来源 | 用户输入 |
| 展示字段 | `classRank`, `gradeRank`, `classTotal`, `gradeTotal` |
| 交互行为 | 数字输入框，可跳过 |
| 必填校验 | 否（显示"跳过"按钮） |

#### 5. 选科组合选择器 (SubjectCombinationSelector)

| 属性 | 规格 |
|-----|------|
| 组件名称 | `SubjectCombinationSelector` |
| 数据来源 | 静态配置 |
| 展示字段 | 语文、数学、英语（必选灰显）+ 物理、化学、地理、政治、生物、历史（多选3门） |
| 交互行为 | 必修三门自动选中不可取消，其余6门选3门 |
| 校验规则 | 必修3门 + 选修必须选满3门 |

#### 6. 考试节点输入 (ExamScheduleInput)

| 属性 | 规格 |
|-----|------|
| 组件名称 | `ExamScheduleInput` |
| 数据来源 | 用户输入 |
| 展示字段 | `examType`, `examDate`, `subjects[]` |
| 交互行为 | 日期选择器 + 类型下拉，可添加多个，可跳过 |
| 考试类型 | 周测、月考、期中、期末 |
| 必填校验 | 否 |

### 交互流程

```
1. 用户进入 /onboarding
   ↓
2. 系统检查是否已有档案
   ├─ 是 → 跳转工作台 /dashboard
   └─ 否 → 显示步骤1
   ↓
3. 步骤1：选择年级
   ├─ 点击年级选项 → 高亮选中
   └─ 点击"下一步" → 校验通过进入步骤2
   ↓
4. 步骤2：选择教材版本
   ├─ 从下拉框选择 → 保存选择
   └─ 点击"下一步" → 进入步骤3
   ↓
5. 步骤3：输入排名（可选）
   ├─ 输入班级/年级排名
   ├─ 点击"跳过" → 直接进入步骤4
   └─ 点击"下一步" → 进入步骤4
   ↓
6. 步骤4：选择选科组合
   ├─ 语数英默认选中（灰显）
   ├─ 从其余6科中选3门
   └─ 选满后"下一步"可点击 → 进入步骤5
   ↓
7. 步骤5：添加考试节点（可选）
   ├─ 点击"添加考试" → 弹出日期+类型选择
   ├─ 点击"跳过" → 直接完成
   └─ 点击"完成建档" → 提交所有数据
   ↓
8. 提交成功
   ├─ 显示"建档完成"动画
   └─ 2秒后自动跳转 /dashboard
```

### 状态定义

| 状态 | 触发条件 | 页面表现 |
|-----|---------|---------|
| 加载中 | 进入页面时 | 骨架屏 |
| 正常 | 数据加载完成 | 显示当前步骤表单 |
| 提交中 | 点击完成建档后 | 按钮 loading，禁用交互 |
| 提交成功 | API 返回成功 | 显示完成动画，自动跳转 |
| 提交失败 | API 返回错误 | Toast 提示错误信息，可重试 |

### 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 网络请求失败 | Toast 提示"网络异常，请重试"，保留已填写数据 |
| 校验不通过 | 高亮错误字段，显示具体错误提示 |
| 已有档案 | 自动跳转工作台，不显示建档流程 |
| 页面刷新 | 从 localStorage 恢复已填写数据 |

### 关联 API

| API | 用途 |
|-----|------|
| `GET /api/v1/config/textbook-versions` | 获取教材版本列表 |
| `POST /api/v1/student/profile` | 创建学生档案 |
| `GET /api/v1/student/profile` | 检查是否已有档案 |

### 核心组件规格

#### 1. 学习模式切换器 (LearningModeSwitch)

| 属性 | 规格 |
|-----|------|
| 组件名称 | `LearningModeSwitch` |
| 数据来源 | API: `GET /api/v1/student/plan/today` |
| 展示字段 | `currentMode: 'workday_follow' | 'weekend_review'` |
| 显示文案 | 工作日跟学 / 周末复习 |
| 交互行为 | 点击展开下拉，选择后调用 API 切换模式 |
| 关联 API | `PATCH /api/v1/student/plan/mode` |

#### 2. 今日计划卡片 (TodayPlanCard)

| 属性 | 规格 |
|-----|------|
| 组件名称 | `TodayPlanCard` |
| 数据来源 | API: `GET /api/v1/student/plan/today` |
| 展示字段 | |
| - 进度 | `completedTasks / totalTasks`（环形进度） |
| - 推荐学科 | `recommendedSubjects[].name` + `reasons[]` |
| - 推断标记 | `isHistoryInferred`（显示警告标记） |
| - 计划状态 | `status: 'generated' | 'in_progress' | 'completed'` |
| 交互行为 | 点击"开始/继续学习"跳转今日计划详情页 |
| 推荐原因标签 | 今日校内同步、今日错误较多、本周反复出错、薄弱知识点待修复、考试临近、本周覆盖不足、最近成绩下滑 |

#### 3. 上传入口卡片 (UploadEntryCard)

| 属性 | 规格 |
|-----|------|
| 组件名称 | `UploadEntryCard` |
| 数据来源 | API: `GET /api/v1/student/material/list?limit=3` |
| 展示字段 | |
| - 主入口 | 拍照上传按钮（最突出） |
| - 次入口 | 文本输入、语音输入 |
| - 最近上传 | `recentUploads[].type`, `uploadTime` |
| 交互行为 | 点击入口跳转上传页面 |

#### 4. 实时答疑卡片 (QAEntryCard)

| 属性 | 规格 |
|-----|------|
| 组件名称 | `QAEntryCard` |
| 数据来源 | API: `GET /api/v1/student/qa/history?limit=3` |

### 核心组件规格

#### 1. 拍照上传组件 (PhotoUploader)

| 属性 | 规格 |
|-----|------|
| 组件名称 | `PhotoUploader` |
| 交互行为 | 点击调用系统相机/相册 |
| 支持格式 | jpg, png, heic |
| 文件限制 | 单张最大 10MB |
| 多图支持 | 是，最多 9 张 |
| 压缩策略 | 上传前压缩至 1080p |

#### 2. 文本输入组件 (TextInputUploader)

| 属性 | 规格 |
|-----|------|
| 组件名称 | `TextInputUploader` |
| 展示形式 | 多行文本输入框 |
| 字数限制 | 最多 2000 字 |
| 支持功能 | LaTeX 公式输入提示 |

#### 3. 语音输入组件 (VoiceInputUploader)

| 属性 | 规格 |
|-----|------|
| 组件名称 | `VoiceInputUploader` |
| 交互行为 | 长按录音/点击开始-停止 |
| 时长限制 | 最长 3 分钟 |
| 转写展示 | 录音完成后显示转写结果 |

#### 4. 上传类型选择器 (UploadTypeSelector)

| 属性 | 规格 |
|-----|------|
| 组件名称 | `UploadTypeSelector` |
| 数据来源 | 静态配置 |
| 类型选项 | 笔记(note)、作业(homework)、练习卷(test)、讲义(handout)、成绩(score) |
| 交互行为 | 单选，必选 |

#### 5. 学科选择器 (SubjectSelector)

| 属性 | 规格 |
|-----|------|
| 组件名称 | `SubjectSelector` |
| 数据来源 | 学生选科组合 |
| 交互行为 | 单选 |
| 自动识别 | OCR 识别后可自动推荐学科 |

#### 6. OCR 状态指示器 (OcrStatusIndicator)

| 属性 | 规格 |
|-----|------|
| 组件名称 | `OcrStatusIndicator` |
| 状态值 | pending/processing/completed/failed |
| 显示样式 | |
| - pending | 灰色等待图标 |
| - processing | 蓝色旋转动画 |
| - completed | 绿色勾选 |
| - failed | 红色叹号 + 重试按钮 |

#### 7. 上传历史列表 (UploadHistoryList)

| 属性 | 规格 |
|-----|------|
| 组件名称 | `UploadHistoryList` |

| 展示字段 | `uploadType`, `subject`, `uploadTime`, `ocrStatus` |
| 默认显示 | 最近 3 条 |

#### 8. 图片裁剪工具 (ImageCropTool)

| 属性 | 规格 |
|-----|------|
| 组件名称 | `ImageCropTool` |
| 触发时机 | 选择/拍摄图片后，进入预览区时可用 |
| 交互行为 | 自由拖拽边框进行裁剪、支持图片旋转 |
| 设计目的 | 引导学生框选单道题目，大幅提高后端 Extraction Agent 的 OCR 准确率与响应速度，降低大模型幻觉 |

### 交互流程

```
1. 用户进入 /upload
   ↓
2. 选择上传方式
   │
   ├─ 拍照上传（主流程）
   │   ├─ 调用相机/相册
   │   ├─ 选择/拍摄图片
   │   ├─ [可选] 使用裁剪工具框选单道题目
   │   └─ 显示在预览区
   │
   ├─ 文本输入
   │   ├─ 展开文本输入框
   │   └─ 输入内容
   │
   └─ 语音输入
       ├─ 开始录音
       ├─ 停止录音
       └─ 显示转写结果
   ↓
3. 选择上传类型
   └─ 必选一个类型
   ↓
4. 选择关联学科
   └─ 可选，OCR 后可自动推荐
   ↓
5. 点击"确认上传"
   ↓
6. 上传处理流程
   ├─ 文件上传到 OSS
   ├─ 触发 OCR 识别
   ├─ 显示识别状态
   │   ├─ 处理中 → 显示进度
   │   ├─ 成功 → 显示识别结果预览
   │   └─ 失败 → 显示错误提示 + 重试/跳过选项
   └─ 更新上传历史
   ↓
7. 上传完成
   ├─ Toast 提示成功
   └─ 可继续上传或返回
```

### 状态定义

| 状态 | 触发条件 | 页面表现 |
|-----|---------|---------|
| 空状态 | 刚进入页面 | 显示上传方式选择 |
| 已选图片 | 选择了图片 | 显示图片预览，可继续添加 |
| 上传中 | 点击确认上传后 | 按钮 loading，禁用交互 |
| OCR 处理中 | 上传成功，等待识别 | 图片上显示处理动画 |
| 上传成功 | OCR 完成 | 显示成功提示，更新历史 |
| OCR 失败 | 识别失败 | 显示错误提示和兜底选项 |

### 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 文件过大 | 提示"文件过大，请压缩后重试" |
| 网络上传失败 | 提示错误，保留已选文件，可重试 |
| OCR 识别失败 | 显示"识别失败"，提供选项：重试 / 手动输入 / 跳过（仍保存原图） |
| 相机权限被拒 | 提示"请在设置中开启相机权限" |

### 关联 API

| API | 用途 |
|-----|------|
| `POST /api/v1/student/material/upload` | 上传学习材料 |
| `GET /api/v1/student/material/list` | 获取上传历史 |
| `GET /api/v1/student/material/{id}/ocr-status` | 查询 OCR 状态 |
| `POST /api/v1/student/material/{id}/retry-ocr` | 重试 OCR |

### 交互流程

```
1. 用户进入 /qa
   ↓
2. 创建新会话或恢复历史会话
   ├─ 新会话 → 显示欢迎消息
   └─ 历史会话 → 加载历史消息
   ↓
3. 用户发送消息
   │
   ├─ 文本输入 + 发送
   │
   └─ 上传题目图片 + 补充说明 + 发送
   ↓
4. 消息发送到后端
   ├─ Routing Agent 识别意图
   ├─ 如有图片 → Extraction Agent OCR
   └─ Tutoring Agent 生成回复
   ↓
5. 流式响应展示
   ├─ 逐字显示 AI 回复
   ├─ 显示答疑策略标签
   └─ 显示关联知识点
   ↓
6. 用户继续对话
   ├─ 回答 AI 提问
   ├─ 继续追问
   └─ 选择"下一步"引导
   ↓
7. 答疑结束
   ├─ Assessment Agent 评估效果
   ├─ 符合条件的题自动进入错题本
   └─ 显示错题沉淀提示
   ↓
8. 下一步选择
   ├─ 巩固练习 → 跳转练习
   ├─ 类似题 → 展示类似题
   └─ 返回 → 回到工作台
```

### 状态定义

| 状态 | 触发条件 | 页面表现 |
|-----|---------|---------|
| 空会话 | 刚创建会话 | 显示欢迎消息，输入框聚焦 |
| 等待回复 | 消息发送后 | 显示"思考中..."动画 |
| 流式输出 | 收到 SSE 数据 | 逐字显示内容 |
| 对话中 | 正常对话 | 显示历史消息和输入框 |
| 网络错误 | 请求失败 | 显示错误提示，可重试 |

### 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 消息发送失败 | 消息显示发送失败图标，可点击重试 |
| 流式响应中断 | 显示"回复中断"，可点击"继续" |
| 图片识别失败 | 提示"图片识别失败"，可手动描述题目 |
| 会话超时 | 提示"会话已超时，已创建新会话" |
| 闲聊拦截 | AI 回复"我主要帮你解答学习问题哦，有什么题目需要帮忙吗？" |

```

   └─ 加载错题列表（默认全部，最新入库）
   ↓
3. 查看错题列表
   ├─ 滚动浏览
   └─ 点击卡片展开详情
   ↓
4. 筛选操作
   ├─ 点击学科 Tab → 筛选该学科错题
   ├─ 选择状态 → 筛选对应状态
   ├─ 选择排序 → 重新排序
   └─ 输入搜索 → 搜索题目/知识点
   ↓
5. 错题操作
   │
   ├─ 点击"查看讲解"
   │   └─ 跳转到讲解详情页
   │
   ├─ 点击"再次召回"（P0）
   │   ├─ 进入召回练习流程
   │   ├─ 完成练习后更新状态
   │   └─ 召回结果影响掌握度

   └─ 点击"开始召回练习"（批量召回）
       ├─ 进入待召回错题队列
       └─ 逐题完成召回
   ↓
6. 召回完成
   ├─ 显示召回结果
   ├─ 更新错题状态
   └─ 显示掌握度变化提示
```

### 状态定义

| 状态 | 触发条件 | 页面表现 |
|-----|---------|---------|
| 加载中 | 进入页面时 | 骨架屏 |
| 正常-有错题 | 数据加载完成 | 显示筛选区+错题列表 |
| 正常-无错题 | 错题为空 | 显示空状态插图 |
| 筛选无结果 | 筛选后无匹配 | 显示"暂无符合条件的错题" |
| 召回中 | 进入召回流程 | 显示召回进度 |

### 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 数据加载失败 | 显示错误提示，可重试 |
| 召回题目加载失败 | 提示错误，可跳过当前题 |
| 召回提交失败 | Toast 提示，本地暂存，后台重试 |

### 关联 API

| API | 用途 |
|-----|------|
| `GET /api/v1/student/errors` | 获取错题列表（支持分页、筛选） |
| `GET /api/v1/student/errors/summary` | 获取错题摘要（各学科数量） |
| `GET /api/v1/student/errors/{id}` | 获取错题详情 |
| `POST /api/v1/student/errors/{id}/recall` | 触发单题召回 |
| `POST /api/v1/student/errors/batch-recall` | 批量召回 |
