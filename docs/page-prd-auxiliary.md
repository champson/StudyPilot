# 辅助页面 PRD 与 UI 设计稿

本文档定义周报/报告页、家长视角页、分享链接页、管理后台的详细页面级需求和 UI 设计描述。

> **技术栈**：Next.js 14+ (App Router) + TailwindCSS + TypeScript  
> **产品定位**：面向上海高中生的 AI 伴学教练，小范围自用（5人以内）  
> **认证方案**：简单 Token，区分学生/家长/管理员

---

## P1. 周报页面（学生视角）

### 页面目标

帮助学生回顾本周学习情况，明确薄弱点和下一步建议，形成持续学习闭环。

### 页面路由

```
/report/weekly
```

### 访问权限

- 学生：可访问自己的周报

### 信息架构

```

│   └── 反复错误点列表
├── 下阶段建议区
│   └── 具体可执行建议列表
└── 底部导航
```

### 核心组件规格

#### 1. 周期选择器 (WeekSelector)

| 属性 | 说明 |
|------|------|
| 数据来源 | 前端计算，基于当前日期 |
| 展示字段 | 周数、日期范围（如：第12周 3.11-3.17） |
| 交互行为 | 点击展开下拉，可选最近 8 周 |
| 默认值 | 当前周（若当前周报未生成，显示上一周） |

#### 2. 核心指标卡片 (MetricCard)

| 属性 | 说明 |
|------|------|
| 数据来源 | `GET /api/v1/student/report/weekly` |
| 展示字段 | `usage_days`、`total_minutes`、`completion_rate` |
| 对比数据 | 与上周对比，显示变化趋势（↑↓→） |
| 状态变化 | 正增长绿色，负增长红色，无变化灰色 |

#### 3. 学科趋势图 (SubjectTrendChart)

| 属性 | 说明 |
|------|------|
| 数据来源 | `student_view_content.subject_trends` |
| 图表类型 | 折线图，展示最近 7 天表现变化 |
| 展示学科 | 最多展示 3 门重点学科 |
| 交互行为 | 桌面端 hover 显示详情，移动端点击显示 |

#### 4. 薄弱点列表 (WeaknessListCard)

| 属性 | 说明 |
|------|------|
| 数据来源 | `student_view_content.high_risk_points`、`student_view_content.repeated_errors` |
| 展示字段 | 知识点名称、所属学科、错误次数/召回状态 |
| 排序规则 | 按风险等级降序，同级按错误次数降序 |
| 最大条数 | 高风险最多 5 条，反复错误最多 5 条 |

#### 5. 建议卡片 (SuggestionCard)

| 属性 | 说明 |
|------|------|
| 数据来源 | `student_view_content.next_stage_suggestions` |
| 展示字段 | 序号、建议内容 |
| 最大条数 | 最多 5 条建议 |

### 交互流程

**正常流程**

1. 学生从工作台点击「周报」入口
2. 系统加载当前周报数据（默认本周，若未生成则显示上一周）
3. 渲染核心指标卡片区
4. 渲染学科趋势图
5. 渲染薄弱点列表
6. 渲染下阶段建议
7. 学生可切换查看历史周报

**切换周期流程**

1. 学生点击周期选择器
2. 下拉展示最近 8 周列表
3. 学生选择目标周
4. 系统请求对应周报数据
5. 刷新页面所有数据展示

### 状态定义

| 状态 | 触发条件 | 页面表现 |
|------|---------|---------|
| 加载中 | 请求周报数据 | 骨架屏，保持布局结构 |
| 正常 | 数据加载成功 | 展示完整内容 |
| 空状态 | 本周无学习记录 | 提示「本周暂无学习记录，开始学习后生成报告」 |
| 错误 | 请求失败 | 提示「加载失败」+ 重试按钮 |

### 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 周报数据未生成 | 显示「报告生成中，请稍后查看」，提供刷新按钮 |
| 网络请求超时 | 显示错误提示 + 重试按钮 |
| 部分数据缺失 | 对应区域显示「暂无数据」，不影响其他区域 |

### 关联 API

| API | 用途 |
|-----|------|
| `GET /api/v1/student/report/weekly?week={week}` | 获取指定周周报（学生视角） |

### 核心组件规格

#### 1. 使用频率卡片组 (UsageMetricsGroup)

| 属性 | 说明 |
|------|------|
| 数据来源 | `GET /api/v1/parent/report/weekly` |
| 展示字段 | `usage_days`、`total_minutes`、`avg_daily_minutes` |
| 对比数据 | 与上周对比，显示变化（文字描述） |

#### 2. 风险摘要卡片 (RiskSummaryCard)

| 属性 | 说明 |
|------|------|
| 数据来源 | `parent_view_content.risk_summary` |
| 展示字段 | 知识点名称、所属学科（仅摘要，不展示详情） |
| 最大条数 | 高风险最多 3 条，反复错误最多 3 条 |
| 特殊说明 | 不展示原始题目图片、完整答疑原文 |

#### 3. 成绩变化表格 (ScoreChangeTable)

| 属性 | 说明 |
|------|------|
| 数据来源 | `parent_view_content.score_changes` |
| 展示字段 | 学科、分数、班级排名变化、年级排名变化 |
| 条件展示 | 仅在有考试成绩数据时展示此区域 |

#### 4. 家长支持建议卡片 (ParentSupportCard)

| 属性 | 说明 |
|------|------|
| 数据来源 | `parent_view_content.parent_support_suggestions` |
| 展示字段 | 建议内容 |
| 最大条数 | 最多 3 条 |

#### 5. 分享链接按钮 (ShareLinkButton)

| 属性 | 说明 |
|------|------|
| 交互行为 | 点击生成分享链接，复制到剪贴板 |
| 链接有效期 | 7 天 |
| 反馈 | 生成成功后显示 Toast 提示「链接已复制」|

### 交互流程

**正常流程**

1. 家长从首页进入「孩子报告」
2. 系统验证家长与学生绑定关系
3. 加载当前周报数据（家长视角）
4. 渲染各内容区域
5. 家长可切换查看历史周报
6. 家长可点击生成分享链接

**生成分享链接流程**

1. 家长点击「生成分享链接」按钮
2. 系统生成带过期时间的签名 URL
3. 自动复制到剪贴板
4. 显示 Toast「链接已复制，有效期 7 天」
5. 家长可将链接分享给其他人

| 登录中 | 提交登录请求 | 按钮 loading，禁用输入 |
| 登录失败 | 验证不通过 | 错误提示，恢复输入 |
| 登录成功 | 验证通过 | 跳转到目标页面 |

### 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| Token 无效 | 表单下方显示「Token 无效，请确认后重试」|
| Token 已过期 | 表单下方显示「Token 已过期，请联系管理员」|
| 账号密码错误 | 表单下方显示「账号或密码错误」|
| 网络错误 | Toast 提示「网络错误，请重试」|

### 关联 API

| API | 用途 |
|-----|------|
| `POST /api/v1/auth/token-login` | Token 登录（学生/家长）|
| `POST /api/v1/auth/admin-login` | 账号密码登录（管理员）|
