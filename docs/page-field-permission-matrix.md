# 页面字段权限矩阵

本文档定义系统中每个页面的字段级访问权限，按角色（学生 / 家长 / 管理员 / 分享链接访客）区分可见性与可操作性。

**角色说明**

| 角色 | 说明 |
|------|------|
| `student` | 登录的学生本人 |
| `parent` | 通过家长 Token 绑定的家长 |
| `admin` | 运营管理员（人工纠偏、模式切换） |
| `share` | 通过分享链接访问的匿名访客 |

**权限符号**

| 符号 | 含义 |
|------|------|
| ✅ | 可见且可操作 |
| 👁 | 仅可见（只读） |
| ❌ | 不可见 / 无权限 |
| `—` | 该角色不访问此页面 |

## 汇总：敏感字段硬性隔离规则

以下字段**任何情况下不出现在分享链接（share 角色）中**：

| 字段 | 归属表 | 原因 |
|------|--------|------|
| `class_rank` / `grade_rank` | `student_profiles`, `exam_records` | 涉及隐私比较 |
| `original_images` (OSS URL) | `study_uploads`, `error_book` | 原始学习材料不对外 |
| `full_qa_history` | `qa_sessions`, `qa_messages` | 完整对话记录仅自用 |
| `structured_summary` | `qa_sessions` | Assessment 内部数据 |
| `admin_notes` / `correction_reason` | `manual_corrections` | 运营内部数据 |
| `content_hash` | `error_book` | 系统内部去重字段，无展示意义 |

上述隔离通过 `app/core/data_masking.py` 中的 `SHARE_OMIT_FIELDS` 集合在序列化层统一过滤（见架构设计 §5.3）。
